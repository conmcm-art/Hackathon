# Content Style Guide

Use this guide for all user-facing text.

## Voice and tone
- Plain, friendly, and direct.
- Explain what happened and what to do next.
- Avoid internal jargon and technical wording.

## Writing rules
- Prefer short sentences.
- Use action-first labels ("Start demo", "Confirm pickup").
- Use consistent naming for roles and flow steps.
- Write error messages with a fix path.

## UI copy patterns
- Buttons: verb-first and specific.
- Empty states: what this area is + how to begin.
- Errors: what failed + how to recover.
- Success: what completed + what comes next.

## Microcopy examples
- Good: "Pickup scheduled for 4:30 PM."
- Better: "Pickup scheduled for 4:30 PM. Track delivery next."

## Accessibility and readability
- Avoid all caps for long labels.
- Avoid emoji-only meaning.
- Keep key instructions readable on small screens.
